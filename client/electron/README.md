# Persona Desktop Shell

Phase 1 Electron wrapper for the Persona VRM avatar. Runs the existing
Vite + Three.js frontend inside a frameless, transparent, always-on-top
window with a system tray icon.

## Quick start

From the project root (Windows / PowerShell):

```powershell
cd electron
npm install
npm run dev
```

`npm run dev` starts the Vite dev server (`http://localhost:5173`) and
launches Electron once Vite is reachable. Use `Ctrl+C` once in the same
terminal to stop both processes.

If you'd rather run the two halves in separate terminals:

```powershell
# Terminal 1 — Vite dev server
cd frontend
npm run dev

# Terminal 2 — Electron shell (after the Vite server says "ready")
cd electron
npm start
```

## File layout

```
electron/
├── main.js                          — main process: window, tray, IPC, lifecycle
├── preload.js                       — context bridge exposing window.personaShell
├── package.json                     — Electron-only dependencies + dev scripts
├── README.md                        — this file
├── assets/
│   └── tray-icon.png                — 32x32 placeholder tray icon (regenerated by postinstall)
├── build/
│   └── icon.png                     — 256x256 app icon for the packaged exe (regenerated by postinstall, gitignored)
└── scripts/
    ├── make-placeholder-icon.js     — writes assets/tray-icon.png from baked base64 (runs on postinstall)
    └── make-app-icon.js             — writes build/icon.png, the 256x256 app/installer icon (runs on postinstall)
```

The frontend itself is unchanged in shape — Electron just loads
`http://localhost:5173` in dev. In packaged builds it loads
`../frontend/dist/index.html`, which electron-builder ships as an
`extraResources` folder at `resources/frontend/dist` — exactly where
`__dirname/../frontend/dist` resolves from inside `app.asar`, so no
path logic changes between dev and prod. See **Packaging** below.

## How dev mode works

- `npm run dev` in `electron/` chains two commands via `concurrently`:
    1. `dev:vite` — runs `npm --prefix ../frontend run dev` so the renderer is served from Vite with hot reload.
    2. `dev:electron` — `wait-on http://localhost:5173 && electron .` waits for Vite to be reachable, then launches the Electron main process pointing at the dev URL.
- The main process detects dev mode when `process.env.NODE_ENV === 'development'`, when `--dev` is passed, or when `app.isPackaged === false` (the default during local development).
- DevTools open detached automatically in dev so they don't crowd the small overlay window.

## Tray icon

- The placeholder is a magenta (`#e879f9`) circle — deliberately distinctive so it's obvious which tray icon belongs to Persona during development.
- It's regenerated by `node scripts/make-placeholder-icon.js`, which runs automatically via the `postinstall` script in `package.json`.
- To install a custom icon, overwrite `electron/assets/tray-icon.png` with your own 32x32 (or larger, ideally 32x32 or 64x64) PNG. **No rebuild needed** — restart Electron and the new icon is picked up.
- If you want a permanent replacement, you can either:
    1. Replace the base64 string in `scripts/make-placeholder-icon.js`, or
    2. Delete the `postinstall` script and commit your icon directly.

## Packaging (Windows installer)

Build a distributable `.exe` with electron-builder:

```powershell
cd electron
npm run dist          # NSIS installer + portable exe -> client/dist-app/
# or
npm run dist:dir      # unpacked app only (no installer), faster for testing
```

`npm run dist` first runs `build:frontend` (`vite build`) and then
electron-builder. Output lands in `client/dist-app/`:

- `Violet Setup <version>.exe` — NSIS installer (lets the user pick the
  install dir, makes Start-menu + desktop shortcuts).
- `Violet-<version>-portable.exe` — single-file portable, no install.

Both are **unsigned** — Windows SmartScreen will warn on first run
("More info → Run anyway"). Code signing needs a certificate; wire it in
later via `CSC_LINK` / `CSC_KEY_PASSWORD`.

### What gets packaged

Only the overlay **client**. The backend stays Dockerized and separate —
the app still talks to `localhost:8000`. On launch `dockerCompose.js`
tries `docker compose up -d`; in a packaged install there's no
co-located `server/` folder, so it logs and skips unless you point it at
one with the `PERSONA_SERVER_DIR` env var. Bring the backend up yourself
(or set that var) for voice/LLM features.

WhatsApp (`whatsapp-web.js` + puppeteer) is `asarUnpack`'d so Chrome can
spawn; its session lives in `userData`. A packaged build on a fresh
machine needs a Chrome available for that feature.

### ⚠️ winCodeSign symlink error on a clean machine

electron-builder downloads a `winCodeSign` bundle that contains macOS
symlinks. Extracting them needs the Windows "create symbolic link"
privilege, so on a stock machine the build fails with:

```
ERROR: Cannot create symbolic link ... libcrypto.dylib
A required privilege is not held by the client.
```

Fix it once, any of:

1. **Enable Developer Mode** (Settings → Privacy & security → For
   developers → Developer Mode = On). Grants the privilege to your user,
   no admin needed. **Recommended.**
2. Run the build from an **elevated** PowerShell ("Run as administrator").
3. Pre-extract the bundle without the macOS dir into the builder cache:
   `…\AppData\Local\electron-builder\Cache\winCodeSign\winCodeSign-2.6.0\`
   (extract the cached `*.7z` with `-xr!darwin`). electron-builder then
   reuses it and skips the failing extraction.

## Tray menu

- **Left-click** the tray icon: toggle the avatar window's visibility.
- **Right-click** the tray icon: context menu with **Show / Hide**, separator, **Quit**.
- Closing the window (e.g., via future UI) **does not** quit the app. Only the tray "Quit" item shuts Persona down — it's designed to live in the background.

## Phase 1 limitations

These are intentional and tracked for later phases:

- **No click-through.** The whole window blocks mouse events so the chat input and resize handles work. Selective click-through (so clicks pass through transparent parts of the avatar canvas onto the desktop) is a Phase 2+ concern.
- **No microphone, no wake word.** The "Persona" wake-word listener and voice input pipeline is Phase 2.
- **No auto-update.** Packaging exists now (see **Packaging** above — `npm run dist` produces an installer + portable exe), but there's no auto-update channel or code signing yet.
- **Placeholder tray icon.** The magenta circle is a clear "replace me" marker; ship a branded icon before Phase 2.
- **No frontend changes besides transparency.** The renderer was switched to a transparent clear, the body background was made transparent, and a 28px drag region was added at the top of the window. Everything else — managers, runtime, chat input, backend client — is untouched.

## Troubleshooting

- **Window appears with a solid black or white background.** The renderer transparency or Windows occlusion workaround regressed. Confirm:
    1. `frontend/js/core/renderer.js` uses `alpha: true` and `setClearColor(0x000000, 0)`.
    2. `frontend/js/core/scene.js` does **not** set `scene.background`.
    3. `app.commandLine.appendSwitch('disable-features', 'CalculateNativeWinOcclusion')` is the first call in `main.js`.
- **Tray icon does not appear.** Verify `electron/assets/tray-icon.png` exists and is a valid PNG (32x32, non-zero bytes). Re-run `node scripts/make-placeholder-icon.js`.
- **Window never opens.** Check that Vite is actually reachable at `http://localhost:5173`. `wait-on` will silently hold the launch until it is.
